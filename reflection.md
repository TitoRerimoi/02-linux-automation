# Reflection

This project provided practical experience with Linux administration, service management, access control, firewall configuration, logging and automation.

The most valuable lesson was understanding idempotent provisioning. Running the script multiple times consistently produced the same desired configuration without causing unintended changes.

The project also reinforced the importance of least privilege through dedicated service accounts, restricted permissions and ACL-based access control.

Using systemd demonstrated how production services are managed, monitored and automatically restarted after failures.

Overall, the project highlighted how infrastructure can be configured in a repeatable, auditable and secure manner using standard Linux tooling.
